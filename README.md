# Pure Start

一个干净、轻量的 Chrome 浏览器新标签页扩展。没有框架，没有追踪，没有乱七八糟的东西。

打开新标签页时你会看到一个简洁的起始页：时间、搜索框、可自定义的书签图标，以及 Bing 每日壁纸作为背景。

## 特性

- 时间与日期显示
- 多引擎搜索栏，支持自定义搜索引擎
- 图标式书签，自动抓取网站 favicon，长按可拖拽排序
- Bing 每日一图作为背景，也支持自定义上传壁纸
- 每日一言，来自 Hitokoto API
- 深色 / 浅色 / 跟随系统三种主题
- 自由布局编辑，模块可以拖放到页面任意位置，有中心吸附辅助对齐
- 所有数据支持导出导入，换浏览器无缝迁移
- 零外部依赖，纯 HTML + CSS + JavaScript

## 安装

1. 下载或克隆本项目到本地
2. 打开 Chrome，进入 `chrome://extensions/`
3. 打开右上角的「开发者模式」
4. 点击「加载已解压的扩展程序」
5. 选择项目文件夹
6. 打开一个新标签页，搞定

## 使用

**日常操作**

- 在搜索框输入关键词，回车搜索
- 点击书签图标打开对应网站
- 按 `/` 键快速聚焦搜索框

**书签管理**

- 长按书签图标进入排序模式，拖拽调整顺序
- 右键书签图标打开菜单，可以编辑或删除
- 在设置面板中可以批量管理书签

**布局编辑**

- 右键页面空白区域打开设置面板
- 点击「编辑页面布局」进入编辑模式
- 拖拽模块手柄或模块本体来调整位置
- 靠近页面水平中心时会出现吸附线和磁吸效果
- 保存、取消或恢复默认布局

**其他**

- 按 ESC 逐层退出弹窗、排序模式或布局编辑
- 在设置中可以导出全部配置（书签、设置、壁纸），在新设备上导入即可还原

## 自定义搜索引擎

在设置面板中进入「管理自定义引擎」，可以添加自己常用的搜索引擎。URL 中使用 `{query}` 作为搜索关键词的占位符。

例如添加 GitHub 搜索：
- 名称：GitHub
- URL：`https://github.com/search?q={query}`

添加后该引擎会出现在搜索栏右侧的切换菜单中。

## 项目结构

```
start_page/
├── manifest.json       # Chrome 扩展配置 (Manifest V3)
├── index.html          # 页面结构
├── style.css           # 样式 (CSS Variables + 毛玻璃 + 自适应壁纸文字)
├── script.js           # 核心逻辑 (Storage / 布局 / 书签 / 设置)
├── icons/              # 扩展图标
└── README.md
```

没有构建工具，没有 node_modules，打开就能用，改完刷新即可看到效果。

## 技术细节

样式使用 CSS Variables 处理主题切换，布局采用 absolute 定位配合百分比坐标，编辑模式下有中心吸附磁吸算法。存储通过 chrome.storage.sync（设置）和 chrome.storage.local（书签和壁纸数据），同时提供 localStorage 降级方案以便在非扩展环境下调试。

壁纸文字可见性使用多层暗色光晕实现，类似视频字幕的渲染方式，在亮色和暗色壁纸上都能看清。拖拽模块时使用 `will-change` 和 `contain` 做性能隔离。
